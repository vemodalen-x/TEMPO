# TEMPO design proposals

Files in this directory are non-authoritative design input. They do not change
the active policy, grant a warrant, implement a capability, or authorize work.

- [`governed-experiment-lane.md`](governed-experiment-lane.md) proposes a
  bounded evidence-only lane between planning and product implementation.
- [`experiment-spec.example.json`](experiment-spec.example.json) illustrates
  the proposed contract without representing an approved experiment.
- [`auditability-ux-phase1.md`](auditability-ux-phase1.md) specifies the first
  audit/retraceability slice and links to a synthetic, read-only interaction
  prototype for founders and small teams.

Implementation requires a separately assessed hypothesis, signed decision and
charter, scoped task, budget, deadline, and current human warrant.
